# Техническое задание: SecureLog
### Цель:
Разработать веб-приложение для регистрации пользователей, логирования их действий и
демонстрации ключевых возможностей PostgreSQL: функции, триггеры, резервное
копирование, права доступа и безопасность.

---
### Стек:
- Frontend:
  - Vanilla JavaScript
  - Fetch API
- Backend:
  - Fastify
  - PostgreSQL
  - Библиотека `pg`
  - Запуск shell-команд через `child_process`
---
### Функционал
Аутентификация
- Регистрация
  - POST `/api/register`
  - Передаёт `name`, `email`, `password`
  - Пароль хешируется через `bcrypt`
  - Хеш сохраняется в БД
  - При регистрации вызывается функция `create_user(name, email, password_hash)`
- Логин
  - POST `/api/login`
  - Принимает `email`, `password`
  - Сравнение с bcrypt-хешем из БД
  - Успешный логин → JWT токен
  - Неуспешный → отказ с 401
- Проверка токена
  - Все приватные маршруты используют middleware проверки JWT
---
### Логирование действий
- Любое действие пользователя (например, изменение имени) записывается в
таблицу logs
- Используется триггер `AFTER UPDATE` на таблицу users
- Пример: изменение `email → logs` получает `user_id`, `action`, `timestamp`
---
### SQL-функции
- `get_user_by_id(id)` — возвращает пользователя
- `recent_logs(user_id)` — последние 5 действий
- `search_users(name_pattern)` — поиск по части имени
- `create_user(...)` — используется при регистрации
- Все функции вызываются из Fastify-роутов через параметризированные запросы
---
### Управление правами
- В БД создать двух пользователей:
  - `admin_user` — доступ ко всем операциям
  - `readonly_user` — только `SELECT`
- Попробовать подключиться под обоими и сравнить поведение
- Настроить `GRANT`, `REVOKE`, роли, схемы
---
### Резервное копирование
- На странице настроек кнопка: **"Сделать бэкап"**
  - Вызывает скрипт с `pg_dump`, сохраняет файл в `/backups/`
- Кнопка: **"Восстановить БД"**
  - Загружает `.sql` файл и восстанавливает БД через `psql`
- Бэкапы хранятся с меткой времени
---
### SQL-инъекции и защита
- Реализовать поиск пользователя по имени `/api/search?name=...`
- Написать уязвимую версию (для демонстрации)
- Затем — исправить с параметризацией
- Подготовить документ с примером атаки и её устранением
---
### Шифрование / HTTPS
- Сервер Fastify работает по HTTPS (локальный сертификат)
- Все данные передаются зашифрованно
- Хранение пароля — только через `bcrypt`
